## ğŸš€ Getting Started

1. **Fork the repository**
   ```bash
   # Click "Fork" on GitHub
   ```

2. **Clone your fork**
   ```bash
   git clone https://github.com/YOUR-USERNAME/prm-agent.git
   cd prm-agent
   ```

3. **Create a branch**
   ```bash
   git checkout -b feature/your-feature-name
   # or
   git checkout -b fix/bug-description
   ```

4. **Make your changes**
   - Follow the existing structure and style
   - Test your changes thoroughly
   - Update documentation if needed

5. **Commit your changes**
   ```bash
   git add .
   git commit -m "Clear description of your changes"
   ```

6. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

7. **Open a Pull Request**
   - Go to the original repository
   - Click "New Pull Request"
   - Select your branch
   - Describe your changes clearly

## ğŸ“ Contribution Guidelines

### Agent Files (.agent.md)
When modifying agent files:
- Maintain the existing structure (Role, Identity, Principles, etc.)
- Keep instructions clear and concise
- Test the agent with actual review data
- Ensure backward compatibility when possible

### Documentation
- Use clear, friendly language
- Include examples where helpful
- Keep it practical and actionable
- Test instructions on a fresh setup

### Code Style
- Follow the existing format
- Use meaningful variable/section names
- Comment complex logic
- Keep it simple and maintainable

### Commit Messages
- Use clear, descriptive messages
- Start with a verb (Add, Fix, Update, Remove)
- Reference issues when applicable
- Examples:
  - `Add support for PDF templates`
  - `Fix Oracle URL fetching bug`
  - `Update README with new examples`

## ğŸ§ª Testing Your Changes

Before submitting:
1. Test with real performance review data
2. Verify all agents work in the workflow
3. Test on at least one platform (Claude Code, Cursor, etc.)
4. Check that documentation matches the changes
5. Ensure no sensitive data is included in commits
